# 在线判题系统技术文档

## 1. 项目概述

### 1.1 系统简介

智码在线判题系统（Code Smart Online Judge System, CSOJ）是一种用于自动评判用户提交的程序代码的平台。本系统旨在为编程教育、算法训练和编程竞赛提供一个完整的解决方案，支持多种编程语言，提供安全的代码执行环境，并实现公平、准确的评判机制。

### 1.2 系统目标

- 提供丰富多样的编程题目供用户练习
- 实现安全可靠的代码执行和判题功能
- 建立积分激励机制，提高用户参与度
- 支持用户间的讨论和交流
- 提供个性化的学习体验和进度跟踪

### 1.3 技术栈概览

- **前端**：Vue 3 + TypeScript + Arco Design
- **后端**：Spring Boot + MyBatis-Plus + Spring Security
- **判题系统**：Docker 容器化代码沙箱
- **数据库**：MySQL
- **搜索引擎**：Elasticsearch（用于讨论区搜索）
- **缓存**：Redis

## 2. 系统架构

### 2.1 整体架构

系统采用前后端分离的架构设计，主要分为以下几个部分：

1. **前端应用**：基于Vue 3的单页面应用，负责用户界面展示和交互
2. **后端服务**：基于Spring Boot的RESTful API服务，处理业务逻辑
3. **代码沙箱**：独立的代码执行环境，负责安全地运行和评估用户提交的代码
4. **判题核心**：评判代码执行结果
5. **数据存储**：MySQL数据库存储系统数据，Redis提供缓存支持

系统可以部署在单机环境或分布式环境中：

- **开发环境**：单机部署，所有组件在同一台服务器上运行
- **生产环境**：分布式部署，前端、后端、代码沙箱、数据库等组件可分别部署在不同服务器上

## 3. 核心功能模块

### 3.1 用户管理模块

#### 3.1.1 功能描述

用户管理模块负责用户的注册、登录、信息管理等功能，是系统的基础模块。

#### 3.1.2 主要功能

- 用户注册与登录
- 用户信息管理
- 用户权限控制
- 用户统计数据展示

#### 3.1.3 关键实现

- 使用Spring Security实现身份认证和授权
- 基于JWT的无状态会话管理
- 用户角色分为普通用户、管理员等不同权限级别

### 3.2 题目管理模块

#### 3.2.1 功能描述

题目管理模块负责编程题目的创建、编辑、展示和管理，是系统的核心内容模块。

#### 3.2.2 主要功能

- 题目创建与编辑
- 题目分类与标签管理
- 题目列表展示与搜索
- 题目详情查看

#### 3.2.3 关键实现

- 支持Markdown格式的题目内容编辑
- 题目难度分级和标签系统
- 题目的测试用例管理
- 题目的判题配置（时间限制、内存限制等）

### 3.3 判题系统模块

#### 3.3.1 功能描述

判题系统是平台的核心技术模块，负责安全、高效地执行用户提交的代码并进行评判。

#### 3.3.2 主要功能

- 代码提交与执行
- 执行结果评判
- 多语言支持
- 安全隔离与资源限制

#### 3.3.3 关键实现

- 基于Docker的代码沙箱实现代码隔离执行
- 支持Java、Python、C++等多种编程语言
- 实现时间限制、内存限制等资源控制
- 采用策略模式处理不同语言的编译和执行
- 安全管理器防止恶意代码执行

### 3.4 积分系统模块

#### 3.4.1 功能描述

积分系统为用户提供激励机制，鼓励用户积极参与平台活动。

#### 3.4.2 主要功能

- 积分规则管理
- 积分获取与消费
- 积分排行榜
- 用户签到

#### 3.4.3 关键实现

- 多种积分获取途径（解题、发布题解、签到等）
- 不同时间维度的排行榜（日榜、周榜、月榜、总榜）
- 积分历史记录追踪

### 3.5 讨论区模块

#### 3.5.1 功能描述

讨论区模块为用户提供交流平台，可以分享经验、讨论题目解法等。

#### 3.5.2 主要功能

- 发布与查看帖子
- 评论与回复
- 点赞与收藏
- 帖子搜索

#### 3.5.3 关键实现

- 基于Elasticsearch的全文搜索
- 帖子分类与标签系统
- 评论的多级展示

## 5. API接口文档

### 5.1 用户接口

#### 5.1.1 用户注册

- **URL**: `/api/user/register`
- **方法**: POST
- **请求参数**:
  ```json
  {
    "userAccount": "账号",
    "userPassword": "密码",
    "checkPassword": "确认密码"
  }
  ```
- **响应结果**:
  ```json
  {
    "code": 0,
    "data": "用户ID",
    "message": "ok"
  }
  ```

#### 5.1.2 用户登录

- **URL**: `/api/user/login`
- **方法**: POST
- **请求参数**:
  ```json
  {
    "userAccount": "账号",
    "userPassword": "密码"
  }
  ```
- **响应结果**:
  ```json
  {
    "code": 0,
    "data": {
      "id": "用户ID",
      "userName": "用户名",
      "userRole": "用户角色",
      "token": "JWT令牌"
    },
    "message": "ok"
  }
  ```

#### 5.1.3 获取当前用户信息

- **URL**: `/api/user/current`
- **方法**: GET
- **请求头**: Authorization: Bearer {token}
- **响应结果**:
  ```json
  {
    "code": 0,
    "data": {
      "id": "用户ID",
      "userName": "用户名",
      "userRole": "用户角色",
      "solvedCount": "已解决题目数",
      "submissionCount": "提交次数",
      "points": "积分"
    },
    "message": "ok"
  }
  ```

### 5.2 题目接口

#### 5.2.1 获取题目列表

- **URL**: `/api/question/list/page`
- **方法**: POST
- **请求参数**:
  ```json
  {
    "pageSize": 10,
    "pageNum": 1,
    "sortField": "createTime",
    "sortOrder": "descend",
    "tags": ["数组", "字符串"],
    "difficulty": "中等"
  }
  ```
- **响应结果**:
  ```json
  {
    "code": 0,
    "data": {
      "records": [
        {
          "id": "题目ID",
          "title": "题目标题",
          "tags": "标签",
          "difficulty": "难度",
          "acceptedNum": "通过数",
          "submitNum": "提交数"
        }
      ],
      "total": "总数"
    },
    "message": "ok"
  }
  ```

#### 5.2.2 获取题目详情

- **URL**: `/api/question/{id}`
- **方法**: GET
- **响应结果**:
  ```json
  {
    "code": 0,
    "data": {
      "id": "题目ID",
      "title": "题目标题",
      "content": "题目内容",
      "tags": "标签",
      "difficulty": "难度",
      "acceptedNum": "通过数",
      "submitNum": "提交数",
      "judgeConfig": {
        "timeLimit": "时间限制",
        "memoryLimit": "内存限制",
        "stackLimit": "堆栈限制"
      }
    },
    "message": "ok"
  }
  ```

#### 5.2.3 提交代码

- **URL**: `/api/question_submit/`
- **方法**: POST
- **请求参数**:
  ```json
  {
    "questionId": "题目ID",
    "language": "编程语言",
    "code": "代码内容"
  }
  ```
- **响应结果**:
  ```json
  {
    "code": 0,
    "data": "提交ID",
    "message": "ok"
  }
  ```

### 5.3 积分接口

#### 5.3.1 获取积分排行榜

- **URL**: `/api/leaderboard/{timeRange}`
- **方法**: GET
- **路径参数**: timeRange - 时间范围（day/week/month/all）
- **响应结果**:
  ```json
  {
    "code": 0,
    "data": {
      "users": [
        {
          "userId": "用户ID",
          "userName": "用户名",
          "points": "积分"
        }
      ]

    "message": "ok"
  }
  ```

## 6. 安全设计

### 6.1 用户认证与授权

- 基于JWT的无状态认证
- 基于角色的访问控制（RBAC）
- 密码加密存储（BCrypt）

### 6.2 代码执行安全

- Docker容器隔离执行环境
- 资源限制（CPU、内存、时间等）
- 安全管理器拦截危险操作（文件读写、网络访问等）
- 代码静态分析检测恶意代码

### 6.3 API安全

- 输入验证和参数校验
- 防SQL注入
- 防XSS攻击
- CSRF防护

## 7. 部署指南

### 7.1 环境要求

- JDK 11+
- Node.js 14+
- MySQL 5.7+
- Redis 6.0+
- Docker 20.10+
- Elasticsearch 7.x（可选，用于讨论区搜索）

### 7.2 后端部署

1. 克隆后端代码仓库
2. 配置application.yml中的数据库连接信息
3. 执行SQL脚本初始化数据库
4. 使用Maven打包：`mvn clean package`
5. 运行JAR包：`java -jar csoj-backend.jar`

### 7.3 代码沙箱部署

1. 克隆代码沙箱仓库
2. 配置Docker环境
3. 使用Maven打包：`mvn clean package`
4. 运行JAR包：`java -jar csoj-code-sandbox.jar`

### 7.4 前端部署

1. 克隆前端代码仓库
2. 安装依赖：`npm install`
3. 配置API地址（.env文件）
4. 构建生产版本：`npm run build`
5. 将dist目录部署到Web服务器（如Nginx）

## 11. 常见问题与解决方案

### 11.1 判题系统相关

- **问题**：判题超时
  **解决方案**：检查代码沙箱资源配置，优化Docker容器参数

- **问题**：判题结果不准确
  **解决方案**：检查测试用例和判题标准，确保一致性

### 11.2 系统部署相关

- **问题**：前后端跨域问题
  **解决方案**：配置CORS或使用代理服务器

- **问题**：数据库连接失败
  **解决方案**：检查数据库配置和网络连接
